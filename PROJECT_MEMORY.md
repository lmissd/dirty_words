# 项目记忆：dirty_words

## 当前项目

这是一个基于树莓派的儿童文明用语提醒项目。用户已经创建了 GitHub 仓库：

```text
https://github.com/lmissd/dirty_words.git
```

## 设备信息

树莓派板子的主机名已经命名为：

```text
yuangungun
```

这个名字会用于后续 SSH 连接、局域网识别、部署脚本和 systemd 服务配置。

## 用户目标

用户买了一块树莓派板子，想用来开发一个可以识别小朋友是否说脏话的设备。如果小朋友说了脏话，设备要及时提醒。

## 功能设想

1. 可以用语音唤醒，或者持续监听环境语音。
2. 可以调用大模型 API 来辅助判断小朋友是否说了脏话。
3. 识别到脏话后，需要提醒小朋友注意文明用语。

## 建议实现方案

最小可行版本先不做复杂唤醒词，而是先实现：

```text
录音 -> 语音转文字 -> 脏话判断 -> 播放提醒
```

后续再增强为：

```text
持续监听 -> 语音活动检测 -> 分段转文字 -> 本地词库初筛 -> 大模型复判 -> 提醒
```

## 技术选择

- Python 作为主开发语言。
- Raspberry Pi OS 作为树莓派系统。
- USB 麦克风或麦克风阵列用于采集语音。
- 小音箱、蜂鸣器、LED 或屏幕用于提醒。
- Git 做本地版本控制。
- GitHub 做远程代码备份和协作。

## 注意事项

- 仅靠烧录 SD 卡和 Python 不够，还需要麦克风、音频输入输出配置、语音转文字服务、脏话判断逻辑和开机自启动配置。
- 如果目标是“说出脏话马上提醒”，持续监听加语音活动检测会比单纯语音唤醒更合适。
- 涉及儿童语音数据时，要尽量减少保存音频，避免上传不必要的隐私内容。
